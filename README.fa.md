# چرخش عکس با هر بار رفرش

یک پروژه‌ی بسیار ساده و بدون وابستگی برای GitHub Pages که با هر بار رفرش صفحه، عکس بعدی را نمایش می‌دهد.

ترتیب هر بازدیدکننده داخل مرورگر خودش و با `localStorage` ذخیره می‌شود؛ بنابراین هر مرورگر یا دستگاه در اولین ورود از عکس اول شروع می‌کند.

## رفتار پروژه

- ورود اول: عکس ۱
- رفرش اول: عکس ۲
- رفرش بعدی: عکس ۳
- بعد از آخرین عکس: بازگشت به عکس ۱
- ترتیب کاربران، مرورگرها و دستگاه‌های مختلف از هم جداست

> این قابلیت در صفحه‌ی GitHub Pages اجرا می‌شود، نه مستقیماً داخل نمایش خود فایل README در GitHub.

## راه‌اندازی

### ۱. دریافت پروژه

مخزن را دانلود یا clone کنید:

```bash
git clone https://github.com/YOUR_USERNAME/refresh-image-rotator.git
cd refresh-image-rotator
```

### ۲. اضافه‌کردن عکس‌ها

عکس‌های خود را داخل پوشه‌ی `images/` قرار دهید.

### ۳. ویرایش `images.js`

مسیر عکس‌ها را به ترتیب دلخواه وارد کنید:

```js
window.ROTATOR_IMAGES = [
  "./images/photo-1.jpg",
  "./images/photo-2.png",
  "./images/photo-3.webp",
];
```

### ۴. فعال‌کردن GitHub Pages

در مخزن GitHub:

1. وارد **Settings** شوید.
2. بخش **Pages** را باز کنید.
3. در **Build and deployment** گزینه‌ی **Deploy from a branch** را انتخاب کنید.
4. شاخه‌ی `main` و پوشه‌ی `/ (root)` را انتخاب کنید.
5. روی **Save** بزنید.

آدرس سایت به این شکل خواهد بود:

```text
https://YOUR_USERNAME.github.io/REPOSITORY_NAME/
```

## بازگرداندن یک کاربر به عکس اول

آدرس صفحه را یک بار با `?reset=1` باز کنید:

```text
https://YOUR_USERNAME.github.io/REPOSITORY_NAME/?reset=1
```

## نحوه‌ی کار

شماره‌ی عکس بعدی در `localStorage` مرورگر ذخیره می‌شود. فقط وقتی عکس با موفقیت بارگذاری شود، شماره‌ی بعدی ثبت می‌شود.

این پروژه به سرور، دیتابیس، فریم‌ورک، نصب پکیج یا مرحله‌ی build نیاز ندارد.

## محدودیت مهم

برای چنین رفتار تعاملی و جداگانه‌ای برای هر بازدیدکننده، نمی‌توان به اجرای جاوااسکریپت در نمایش عادی README تکیه کرد. همچنین GitHub تصاویر README را از مسیر پراکسی ناشناس خود ارائه می‌کند. به همین دلیل، بخش تعاملی با GitHub Pages اجرا می‌شود و README فقط راهنما و لینک پروژه است.
